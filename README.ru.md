## Об EasyTaxes

Это [сервис онлайн-бухгалтерии в Армении](https://easytaxes.am). Автоматизированный процесс обработки инвойсов, банковских выписок, налоговых отчётов и взаимодействия с налоговой делает сервис отказоустойчивым и доступным по цене. Это оставляет нам больше времени чтобы оказывать лучший сервис за меньшие деньги.

## Что такое EasyTaxes API?

ET API даёт программный доступ к сервису EasyTaxes. Можно получать информацию о компаниях, скачивать налоговые отчёты и документы, загружать инвойсы и файлы — всё автоматически, без входа в приложение. Включая ограниченный доступ к данным налоговой системы налогоплательщика Армении на file-online.taxservice.am.

Используйте ET API для подключения к вашим техническим системам, мониторинга или LLM-моделям.

Изменение статусов компаний и критических данных через API недоступно — такие операции выполняются только через приложение EasyTaxes.

### Кому доступен API

API доступен клиентам с активной подпиской на сервис EasyTaxes. Чтобы получить доступ, необходимо:

1. Иметь активную подписку на EasyTaxes
2. Подать заявку на участие в бета-тестировании через [easytaxes.am/contact](https://easytaxes.am/contact)
3. Дождаться подтверждения и получить персональный API-ключ

## Аутентификация

Все запросы к API требуют аутентификации с помощью персонального API-ключа.

**Как получить API-ключ:**
Войдите в личный кабинет EasyTaxes → раздел «Настройки» → вкладка «API» → нажмите «Создать API-ключ». Каждый клиент получает уникальный персональный ключ. Ключ отображается один раз — сохраните его в надёжном месте.

**Как использовать API-ключ:**
Передавайте ключ в заголовке каждого запроса:

```text
Authorization: Bearer YOUR_API_KEY
```

**Пример запроса:**

```bash
curl -X GET "https://open-api.easytaxes.am/v1/companies" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

## Лимиты запросов

Для равномерного распределения нагрузки используется алгоритм **token bucket**.

| Период   | Лимит       | Интервал  | Всплеск (burst) |
| -------- | ----------- | --------- | --------------- |
| 1 минута | 60 запросов | 1 секунда | 10 запросов     |

**Период** — временной интервал, в течение которого можно отправить максимальное количество запросов.

**Лимит** — максимальное количество запросов за период. За одну минуту можно отправить до 60 запросов. Запросы рекомендуется равномерно распределять по времени.

**Интервал** — рекомендуемая пауза между запросами: 60 сек / 60 запросов = **1 секунда**. Соблюдайте интервал, чтобы избежать ошибок.

**Всплеск (burst)** — максимальное количество запросов, которые можно отправить одновременно без интервальных пауз. Текущее доступное значение всплеска возвращается в заголовке ответа `X-Ratelimit-Remaining`.

### Заголовки ответа

`X-Ratelimit-Remaining` — количество запросов, доступных прямо сейчас без паузы. Уменьшается на 1 после каждого запроса. При значении `0` следующий запрос без задержки вернёт ошибку `429`. Значение восстанавливается автоматически со временем.

При превышении лимита API возвращает ошибку `429 Too Many Requests`. Следующий запрос можно выполнить только после ожидания. Для определения времени ожидания используйте заголовки ответа:

| Заголовок           | Описание                                                    |
| ------------------- | ----------------------------------------------------------- |
| `X-Ratelimit-Retry` | Через сколько секунд можно повторить запрос                 |
| `X-Ratelimit-Limit` | Максимальное значение всплеска, которое будет восстановлено |
| `X-Ratelimit-Reset` | Через сколько секунд всплеск восстановится до максимума     |

**Пример ответа при превышении лимита:**

```text
HTTP/1.1 429 Too Many Requests
X-Ratelimit-Retry: 3
X-Ratelimit-Limit: 10
X-Ratelimit-Reset: 45
```

## Форматы данных

* Все запросы и ответы используют формат **JSON**
* Поддерживаемые форматы загружаемых файлов: `PDF`, `XLSX`, `DOCX`, `PNG`, `JPG`, `WebP`, `HEIC`
* Кодировка: `UTF-8`
* Дата и время передаются в формате `ISO 8601` (`YYYY-MM-DDTHH:MM:SSZ`)
